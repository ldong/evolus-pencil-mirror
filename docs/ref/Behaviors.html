Behavior Reference

To help Stencils developersdefine how the content in shapes is changed based on attributes, Pencil provides Behavior concept. With a Behavior, one or more attributes of an target object can be changed based on Behavior input value.The target object can be a SVG object or HTML object based on the attributes that target object includes.

This document lists all supported Behavior in Pencil, each with both JavaScript syntax and XML syntax for use in stencil coding.

Each target object is set behaviors by a general syntax:

    <For ref="target object">
      <Behavior_1>....</Behavior_1>
      <Behavior_2>....</Behavior_2>
      .......
      <Behavior_n>....</Behavior_n>
    </For>

Suppose there is a Dimension property type variable is used in the whole this document:

    <Property name="box" type="Dimension">150,150</Property>

CustomStyle

This behavior is used to assign value to a specific XML attribute of the target object. Value and name are specified in the input arguments of the behavior.

Target object:

All kind of objects.

XML Syntax:

  <CustomStyle>
    <Arg>propertyName</Arg>
    <Arg>value</Arg>
  </CustomStyle>

Input value:

propertyName: name of property.

value: value is assigned to propertyName.

Result:

The "propertyName" attribute of the target object is assigned value.

Example:

    <CustomStyle>
      <Arg>"width"</Arg>
      <Arg>$box.w</Arg>
    </CustomStyle>

Attr

This behavior is used to assign value to a specific XML attribute of the target object. Value and name (and optional namespace URI) are specified in the input arguments of the behavior. CustomStyle and Attr behavior are used the same. But with Attr, an attribute can be derived from a namespace.

Target object:

All kind of objects.

XML Syntax:

    <Attr>
      <Arg>propertyName</Arg>
      <Arg>value</Arg>
      <Arg>namespace</Arg>
    </Attr>

Input value:

propertyName: name of property.

value: value is assigned to propertyName.

namespace: namespace that property is derived from. If a property have no namespace, the namespace argument can be omited.

Result:

The "propertyName" attribute of the target object is assigned value.

Example: 
    <Attr>
      <Arg>"width"</Arg>
      <Arg>$box.w</Arg>
    </Attr>
    <Attr>
    <Arg>"xlink:href"</Arg>
    <Arg>value</Arg>
    <Arg>PencilNamespaces.xlink</Arg>
    </Attr>

If the namespace was defined in parent node, namespace argument could also be omited.

    <Attr>
      <Arg>"xlink:href"</Arg>
      <Arg>value</Arg>
    </Attr>

Box

This behavior is used to assign value to width and height attributes of the target object.

Target object:

All objects have width and height properties.

XML syntax:

    <Box>DimensionValue</Box>

Input value:

DimensionValue: the value is dimension Type.

Result:

The width and height attributes of the target object is set value based on DimensionValue object.

Example:

Define a Dimension type variable:

    <Property name="box" displayName="Box" type="Dimension">50,50</Property>

then put to Box bevavior:

    <Box>$box</Box>

Can also put a Dimension instance is created in behavior directly:

      <Box>new Dimension(50,50)</Box>

Radius

This behavior sets the "rx" and "ry" attribute of the SVG objects that support corner radius including Rectangle and Ellipse.

Target object:

rectangle and ellipse

XML syntax:

    <Radius>
      <Arg>rx</Arg>
      <Arg>ry</Arg>
    </Radius>

Input value:

rx: radius on horizontal

ry: radus on vertical

Result:

The target object (Rectangle, Ellipse) is set "rx" and "ry".

Example:

    <Radius>
    <Arg>5</Arg>
    <Arg>5</Arg>
    </Radius>

Fill

This behavior sets the "fill" and "fill-opacity" attribute of the target SVG objects that can be filled color.

Target object:

SVG objects that can be filled color.

XML syntax:

    <Fill>color</Fill>

Input value:

color: a color Property type object.

Result:

The target object color and opacity are set.

Example:

    <Fill>Color.fromString("#ffffffff")</Fill>
      <Property name="color" displayName="fColor" type="Color">#000000ff</Property>     
      ....
      <Fill>$color</Fill>

BoxFit

This behavior is used to set text bound and alignment.

Target object:

Only text object.

XML syntax:

    <BoxFit>
      <Arg>bound</Arg>
      <Arg>alignment</Arg>
    </BoxFit>

Input value:

bound: a Bound Property type object.

alignment: an Alignment Property type object.

Result:

The text bound is created and is aligned on horizontal & vertical in this bound.

Example:

    <BoxFit>
      <Arg>Bound.fromBox($box)</Arg>
      <Arg>new Alignment(1,1)</Arg>
    </BoxFit>
    <Property name="textAlign" displayName="Text Alignment" type="Alignment">1,1</Property>
    ...
    <BoxFit>
      <Arg>Bound.fromBox($box)</Arg>
      <Arg>$textAlign</Arg>
    </BoxFit>

Font

This behavior is used to set font of the target object. With this behavior, many attributes of the target object help to display text are assigned value.

Target object:

SVG Text object and HTML objects.

XML Syntax:

    <Font>font</Font>

Input value:

font: a Font Property type object.

Result:

"font-family", "font-size", "font-weight", "font-style" and "text-decoration" attribute of text object are assigned value.

Note that "text-decoration" attribute is only belonged to HTML objects. So it is impossible to set "text-decoration" to SVG Text object.

Example:

    <Font>Font.fromString("Helvetica|normal|normal|14px")</Font>
    <Property name="font" type="Font" displayName="Default Font">Helvetica|normal|normal|14px</Property>
    ...
    <Font>$font</Font>

D

This behavior is used to set "d" attribute of SVG path object. The values are put into D behavior will be listed in input value.

Target object:

Only path object.

XML Syntax:

    <D>[...]</D>

Input value:

[...] : an array of direction instruction functions. The functions includes:

M(x,y) : set point.

L(x,y) : draw a line from a point to x,y.

      Ex: <D>[M(0, 0), L(10,10)]</D>

C(x1, y1, x2, y2, x, y): the same as C in SVG.

c(x1, y1, x2, y2, x, y): the same as c in SVG.

S(x2, y2, x, y): the same as S in SVG.

s(x2, y2, x, y): the same as s in SVG.

Q(x1, y1, x, y): the same as Q in SVG.

q(x1, y1, x, y): the same as q in SVG.

z: the same as z in SVG.

Result:

"d" attribute of path object is assigned value. The array of direction instruction functions and then is transfer to SVG direction instruction.

Example:

      <D>[M(0, 0), L($box.w, 0), L($box.w, $box.h), L(0, $box.h), z]</D>

Transform

This behavior is used to assigned to "transform" attribute of SVG target objects. The new value will be appended to the current value.

Target object:

All svg object.

XML Syntax:

    <Transform>[...]</Transform>

Input value:

[...] : an array of instruction functions. The functions are the same as the SVG functions and include:


rotate(x)

translate(x, y)

scale(x, y)

skewX(a)

skewY(a)

Result:

"transform" attribute of SVG target object is assigned value.

Example:

      <Transform>[scale($box.w.120, $box.h/100), transform(50, 70)]</Transform>

Scale

This behavior is used to assigned to "scale" function of "transform" attribute of SVG object.

Target object:

All svg object.

XML Syntax:

      <Scale>(width_ratio, height_ratio)</Scale>

Input value:

width_ratio: the ratio of the current width to the new width.

height_ratio: the ratio of the current height to the new height.

Result:

The svg object width and height will be changed based on the ratio. Note that using this behavior will empty the current value of "transform" attribute.

Example

    <Scale>
      <Arg>$box.w/120</Arg>
      <Arg>$box.h/100</Arg>
    </Scale>
