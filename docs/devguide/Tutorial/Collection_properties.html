2. Collection properties


Shapes in a collection tend to have the same style such as same font, color, stroke style, â€¦ To be convenient to Pencil Users in using other style, Pencil supports grouping all shapes common style to collection style. Then if collection style is changed, all shapes refer to collection style will be changed, too.


Have a look at this sample. Collection properties are defined and then used in shape properties.


<Shapes xmlns="http://www.evolus.vn/Namespace/Pencil"

xmlns:p="http://www.evolus.vn/Namespace/Pencil"

xmlns:svg="http://www.w3.org/2000/svg"

xmlns:sodipodi="http://sodipodi.sourceforge.net/DTD/sodipodi-0.dtd"

xmlns:inkscape="http://www.inkscape.org/namespaces/inkscape"

id="collection" displayName="My Collection"

description="My First Collection"

author="author">

<Properties>

<PropertyGroup name="Text">

<Property name="defaultTextFont" type="Font" displayName="Default Font">Arial,sans-serif|normal|normal|13px</Property>

<Property name="defaultTextColor" type="Color" displayName="Default Text Color">#000000ff</Property>

</PropertyGroup>

</Properties>

<Shape id="helloworld" displayName="Hello World" icon="Icons/plain-text.png">

<Properties>

<PropertyGroup name="Text">

<Property name="label" displayName="Label" type="PlainText">Hello World</Property>

<Property name="textColor" displayName="Color" type="Color"><E>$$defaultTextColor</E></Property>

<Property name="textFont" displayName="Font" type="Font"><E>$$defaultTextFont</E></Property>

</PropertyGroup>

</Properties>

<Behaviors>

...

</Behaviors>

<p:Content xmlns:p="http://www.evolus.vn/Namespace/Pencil" xmlns="http://www.w3.org/2000/svg">

<text id="text"/>

</p:Content>

</Shape>

</Shapes>


A value is assign to each property is default a string. To get collection properties value for shapes properties, the collection properties value must be put into <E></E> because this value is an expression.
